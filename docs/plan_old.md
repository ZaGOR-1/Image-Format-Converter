# План реалізації Image Format Converter MVP v1

Цей документ описує поетапний план реалізації Python CLI-проєкту **Image Format Converter** на основі вимог із `docs/prompt.md` та `AGENTS.md`.

Мета MVP v1: створити локальну консольну програму для пакетної конвертації RAW-файлів камер і звичайних форматів зображень без GUI, веб-сервера, бази даних або зайвих фреймворків.

## Загальні принципи реалізації

- Використовувати Python 3.12+.
- Розділити відповідальність між CLI, скануванням файлів, побудовою шляхів, конвертерами та результатами виконання.
- Не розміщувати всю логіку в `main.py`.
- Використовувати `pathlib.Path` для шляхів.
- Використовувати type hints у публічних функціях і методах.
- Обробляти помилки окремих файлів без зупинки всієї пакетної обробки.
- Писати код так, щоб у майбутньому можна було додати audio, video, document converters і GUI.

## Фаза 7. Конвертер звичайних зображень

### Ціль

Реалізувати конвертацію між JPG, JPEG, PNG, WEBP, TIF і TIFF через Pillow.

### Задачі

- У `app/converters/image_converter.py` створити `ImageConverter`.
- Підтримати входи:
  - `.jpg`;
  - `.jpeg`;
  - `.png`;
  - `.webp`;
  - `.tif`;
  - `.tiff`.
- Підтримати виходи:
  - `jpg`;
  - `jpeg`;
  - `png`;
  - `webp`;
  - `tif`;
  - `tiff`.
- Після відкриття звичайного зображення застосувати автоповорот по EXIF Orientation:

```python

image = ImageOps.exif_transpose(image)
```

Це базова нормалізація орієнтації, а не розширене керування EXIF.
- Для JPEG/JPG:
  - конвертувати зображення в `RGB`;
  - якщо є alpha-канал, накладати зображення на білий фон;
  - застосовувати `quality`.
- Для WEBP застосовувати `quality`.
- Для PNG зберігати прозорість, якщо це можливо.
- Реалізувати resize:
  - тільки ширина;
  - тільки висота;
  - fit inside box, якщо задано і ширину, і висоту;
  - без спотворення пропорцій.

### Результат фази

Звичайні формати зображень коректно конвертуються між собою, включно з `.tif`, `.tiff`, EXIF auto-rotate, alpha-каналом, JPEG/WebP quality та resize.

## Фаза 15. Критерії готовності

MVP v1 вважається готовим, коли:

- CLI стартує без помилок.
- Один файл конвертується успішно.
- Папка конвертується успішно.
- Рекурсивна обробка працює.
- `--keep-structure` зберігає структуру папок.
- JPG, JPEG, PNG, WEBP, TIF і TIFF конвертуються між собою.
- RAW -> JPG/PNG/TIF/TIFF реалізовано через `rawpy`.
- RAW -> WEBP не дозволяється в MVP v1.
- EXIF Orientation для звичайних зображень нормалізується через `ImageOps.exif_transpose`.
- Alpha-канал PNG коректно прибирається при конвертації в JPG.
- Існуючі файли обробляються через overwrite або unique naming.
- Пакетна обробка не падає через один проблемний файл.
- Підсумок показує total, converted, failed, skipped і список помилок.
- README пояснює встановлення, запуск, аргументи, обмеження та roadmap.
- Тести для scanner, path utils і PNG alpha -> JPG image converter case проходять.

## Ризики та рішення

| Ризик | Рішення |
| --- | --- |
| `rawpy` може складно встановлюватися на деяких системах | Документувати залежність у README, тримати RAW-логіку ізольованою в `RawConverter` |

16. За бажанням додати `pyproject.toml`, якщо потрібна більш формальна структура пакета.

## Що не входить у MVP v1

- GUI.
- Drag-and-drop.
- Audio conversion.
- Video conversion.
- Document conversion.
- Web API або web server.
- База даних.
- Черга задач із persisted history.
- Пресети конвертації.
- Розширене керування EXIF.

Ці можливості можна додавати після стабілізації CLI MVP.
